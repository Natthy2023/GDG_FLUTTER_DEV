# Week 3

## Overview

In Week 3, the focus was on **Dart collections** and how they can be used to efficiently store, organize, and manage data. We explored **List**, **Set**, and **Map**, along with **Dart null safety** to safely handle nullable values and prevent runtime errors.

## Topics Covered

* Lists in Dart
* Sets in Dart
* Maps in Dart
* Differences between List, Set, and Map
* Removing duplicates using Set
* Key-value data storage with Map
* Set operations (difference and intersection)
* Basics of Dart null safety
* Using `stdin` for user input

## Key Learnings

* Lists allow duplicate values and maintain insertion order
* Sets store only unique values and support powerful set operations
* Maps organize data using key-value pairs
* Null safety helps prevent errors caused by null values
* Set difference is useful for identifying missing or unexpected data
